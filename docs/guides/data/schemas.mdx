Schemas are the data types of the Sonr, adding structure to data stored on the network. Schemas are ubiquitous, meaning you're not limited to using the Schemas you created. Check out [Speedway](#) for information on viewing schemas.

## Creating a Schema

<SupportedClients list={["c"]} />

To create a schema, run the following command using the Speedway CLI.

```bash
speedway schema create
```

## Fetching Schemas

<SupportedClients list={["f", "g"]} />

If you know a schema's DID, you have all you need to get it.

<Tabs groupId="platform">
  <TabItem value="flutter" label="ðŸ¦ Flutter">

Querying for a Schema requires a single string value which corresponds to the [Sonr DID Spec](https://github.com/sonr-io/sonr/blob/dev/docs/architecture/1.md)

```dart
Future<QueryWhatIsResponse?> querySchema(String q) async {
  return await MotorFlutterPlatform.instance.querySchema(QueryWhatIsRequest(did: q));
}
```

  </TabItem>
  <TabItem value="go" label="ðŸ¹ Go">

Querying for a Schema requires a single string value which corresponds to the [Sonr DID Spec](https://github.com/sonr-io/sonr/blob/dev/docs/architecture/1.md)

```go
func GetSchema(instance *motorNodeImpl, schemaDid string) {
  resp, err := motorInstance.QueryWhatIsByDid(schemaDid)
  if err != nil {
    // handle error
  }

  // Resturns the schema referenced by the `WhatIs`
  schema := resp.GetSchema()
}
```
